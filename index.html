Merchant Accessibility Desk

Accessibility
Friction Overlay

This version is designed as a merchant-facing control room. No database, no Redis, and no long-running Node server are required in production. The admin page is the main product surface for business owners, while the storefront script is a lightweight collector used for merchant-run QA walkthroughs and preview drills.

Deployment Model

Store: demo-store

Runtime: Static HTML, JS, and CSS on any HTTPS host.

Tradeoff: No cross-device live dashboard stream without a backend.

Settings

Storefront Tracking Controls

These settings are saved only in this browser so a business owner can generate a deployment snippet and run preview drills without a server-side settings API.

Deploy

Paste-Ready Storefront Loader

Option A: Paste this into Ecwid Admin → Website → Design → Custom JavaScript code. The snippet includes its own <script> tags — paste it exactly as shown.
Option B: For published apps, set customJsUrl in your app's details at #develop-apps instead. See the Deployment Guide for details.
